## Technical Context

### Technology Stack
- **Framework**: Next.js 16+ (App Router)
- **Language**: TypeScript (FR-013, Constitution VII)
- **Styling**: Tailwind CSS
- **Authentication**: Better Auth (Client-side) (FR-001, FR-003)
- **Package Manager**: npm

### External Dependencies
- `better-auth`: For handling client-side authentication and JWT generation.
- `lucide-react`: For UI icons.
- `shadcn/ui`: (Assumed) For standardized UI components like Modals, Buttons, Inputs.

### Key Constraints
- **JWT Attachment**: Every `/api/*` request must include `Authorization: Bearer <token>` (FR-004).
- **Shared Secret**: Use `BETTER_AUTH_SECRET` for JWT consistency (FR-005).
- **Performance**: Auth response time < 200ms (SC-001).

## 4. API Integration Plan

### Section Name: API Integration Plan
**Decision**: Create a centralized `lib/api.ts` client using `fetch` with interceptor-like logic for JWT.
**Reasoning**: Ensures all requests are authenticated consistently (FR-004) and handles errors globally.
**Linked Requirements**: FR-004, FR-005, FR-013

- **Base URL**: `process.env.NEXT_PUBLIC_API_URL`
- **Mocking**: Use a simple conditional toggle or MSW if backend is absent.

**Function Signatures**:
- `getTasks(userId: string): Promise<ApiResponse<Task[]>>`
- `createTask(userId: string, data: TaskFormData): Promise<ApiResponse<Task>>`
- `updateTask(userId: string, taskId: number, data: TaskFormData): Promise<ApiResponse<Task>>`
- `deleteTask(userId: string, taskId: number): Promise<ApiResponse<void>>`
- `toggleComplete(userId: string, taskId: number): Promise<ApiResponse<Task>>`

## 5. Better Auth Integration Plan

### Section Name: Better Auth Integration
**Decision**: Initialize Better Auth in `lib/auth.ts` and use it for session management.
**Reasoning**: Leverages established library for secure JWT handling.
**Linked Requirements**: FR-003, FR-004, FR-005, FR-007

- **Initialization**: `createAuthClient({ baseUrl: process.env.BETTER_AUTH_URL })`
- **JWT Extraction**: Extract token from `auth.getSession()` and store in memory/cookie for `api.ts`.
- **Route Protection**: Use Next.js Middleware (`middleware.ts`) to check for valid session tokens.

## 6. State Management Plan

### Section Name: State Management
**Decision**: Use React `useState` and `useEffect` with custom hooks (`useTasks`) for local state; Auth state via Better Auth context.
**Reasoning**: Keeps the app simple and focused (YAGNI) while providing clean abstractions.
**Linked Requirements**: FR-008, FR-009, FR-012

- **Task List**: Managed in `useTasks` hook with optimistic UI updates.
- **Auth Flow**: `useAuth` hook providing `user`, `loading`, `login`, `logout`.

---

## 7. Error Handling Strategy

### Section Name: Error Handling
**Decision**: Map HTTP status codes and backend errors to user-friendly strings.
**Reasoning**: Prevents leaking technical details and improves UX (SC-005).
**Linked Requirements**: FR-002, SC-005

| Error Scenario | Mapping |
|----------------|---------|
| Duplicate Email (409) | "Email already registered" |
| Invalid Credentials (401) | "Invalid email or password" |
| Expired Token (401) | Redirect to `/login` |
| Network Failure | "Something went wrong, please check your connection" |
| Task Not Found (404) | "Task no longer exists" |

## 8. TypeScript Interfaces

### Section Name: TypeScript Interfaces
**Decision**: Define central types in `types/index.ts`.
**Reasoning**: Ensures type consistency across the frontend.
**Linked Requirements**: Constitution VII

```typescript
export interface User {
  id: string;
  email: string;
  name?: string;
}

export interface Task {
  id: number;
  user_id: string;
  title: string;
  description?: string;
  completed: boolean;
  created_at: string;
  updated_at: string;
}

export interface TaskFormData {
  title: string;
  description?: string;
}

export interface ApiResponse<T> {
  data?: T;
  error?: string;
}
```
