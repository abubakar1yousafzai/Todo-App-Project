# Evolution of Todo (Phases 2 & 3) Constitution

## Core Principles

### I. Spec-Driven Development (SDD)
No agent is allowed to write code until the specification is complete and approved. Every implementation step must map back to an explicit requirement in the `specs/` directory.

### II. AI-Generated Implementation (No Manual Coding)
The engineer acts as a System Architect. All code must be generated by AI agents (Claude Code) based on refined specifications. Manual coding is strictly prohibited; the focus is on refining the Spec until the agent generates the correct output.

### III. Stateless Multi-User Architecture
The system must be stateless and support multiple users. Every API request must be authenticated via JWT. Data isolation is non-negotiable; all database operations must be filtered by the authenticated `user_id`.

### IV. Agentic Architecture (Phase 3)
The AI Chatbot must utilize a Model Context Protocol (MCP) server architecture. All task operations must be exposed as MCP tools. The AI agent (OpenAI Agents SDK) must use these tools to perform actions on behalf of the user.

### V. Conversation Persistence
Chat sessions must be stateless on the server but persistent in the database. Every message must be stored with its `conversation_id` and `user_id`.

### VI. Single Source of Truth
The `specs/` directory is the authoritative source for requirements, architecture, and tasks. If a conflict arises, the hierarchy is: Constitution > Specify > Plan > Tasks.

### VII. Clean Code & Type Safety
Implementation must use modern, type-safe patterns: TypeScript for the frontend and Python with SQLModel/Pydantic for the backend.

## Technology Stack

### Frontend
- **Web App**: Next.js 16+ (App Router), Tailwind CSS
- **Chat Interface**: OpenAI ChatKit
- **Auth**: Better Auth (Client-side)

### Backend
- **Framework**: Python FastAPI (3.13+)
- **AI Framework**: OpenAI Agents SDK
- **MCP**: Official MCP Python SDK
- **ORM**: SQLModel
- **Database**: Neon Serverless PostgreSQL
- **Auth Verification**: JWT validation (Shared secret with Better Auth)

### Infrastructure & Tools
- **Package Management**: `npm` (Frontend), `uv` (Backend)
- **Spec Management**: Spec-Kit Plus
- **AI Agent**: Claude Code

## Development Workflow

1. **Specify (WHAT)**: Define requirements, user journeys, and acceptance criteria in `specs/features/`.
2. **Plan (HOW)**: Design components, API contracts, and database schemas in `specs/plan/`.
3. **Tasks (BREAKDOWN)**: Create atomic, testable work units in `specs/tasks/`.
4. **Implement (CODE)**: Execute code changes referencing Task IDs. No task = No code.

## Governance
- This Constitution supersedes all other development practices.
- Every code file must ideally link back to a Task or Spec section.
- Agents must stop and request clarification if a requirement is underspecified.
